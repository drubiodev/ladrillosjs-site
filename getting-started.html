Getting started with LadrillosJS

A lightweight, zero-dependency web component framework. Build reactive UIs with plain .html files — no JSX, no virtual DOM, no compiler required. You'll have your first component running in about five minutes.

3. The mental model

If you've used React, Vue, or Svelte, here's how LadrillosJS thinks differently.

  • HTML files are the components. No JSX, no SFC compiler, no template strings.
  • Reactivity is just let. Reassign a variable; the bindings re-render. That's the whole API.
  • The browser does the work. Components register as native custom elements with Shadow DOM by default.
  • Expressions in { }. Any JavaScript expression works inside curly braces.

4. Directives cheat sheet

Directives are special attributes prefixed with $. They handle the things HTML can't do on its own.

$if / $else-if / $else

Conditional render

Add or remove an element from the DOM based on a condition.

$show

Toggle visibility

Hide via CSS display: none while keeping the element in the DOM.

$for

Render a list

Loop with optional index and a $key for efficient diffing.

$bind

Two-way binding

Sync form inputs with state — works with text, checkbox, select, textarea.

$ref

Element reference

Get direct DOM access via $refs.name for canvas, focus, etc.

$key

List optimization

Pair with $for to keep DOM nodes stable across reorders.

8. What's new in v2

ESM-only Tree-shakeable Lazy strategies $use() helper configure() Parallel registration
  • Granular entry points — import only what you use: ladrillosjs/core, ladrillosjs/lazy, ladrillosjs/events.
  • Five lazy strategies built in: lazyOnVisible, lazyOnIdle, lazyOnInteraction, lazyOnMedia, lazyOnDelay.
  • Parallel registerComponents() fetches all component files concurrently and reports per-component success/failure.
  • $use("./path.html") infers the tag name from the file path — handy for quick scripts.
  • configure() for cache size and a global error handler hook.
  • LRU component cache (default 25) keeps repeat instantiation fast.

9. Common gotchas

  • Tag name needs a hyphen. Custom elements must contain at least one - (e.g. my-counter, not counter).
  • Use let, not const. Reactivity tracks variable reassignment — const can't be reassigned.
  • Reassign arrays/objects to trigger updates. Use arr = [...arr, item] instead of arr.push(item).
  • Shadow DOM is on by default. Global CSS won't reach inside. Pass useShadowDOM: false when registering if you want it to.
  • ESM-only. Always use <script type="module">. Older browsers need a build step.
